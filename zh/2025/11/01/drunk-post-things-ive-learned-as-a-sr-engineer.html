醉酒帖子：我作为高级工程师学到的东西

2025-11-01

转载: Drunk Post: Things I’ve learned as a Sr Engineer
作者: flipstables

我喝醉了，估计明天醒来会后悔发了这个。但不管了，以下是我作为工程师十年来的醉酒心得排名：

  • 我职业发展最快的方式就是跳槽。
  • 技术栈其实没那么重要，因为在我这个领域（数据领域），软件工程基本就那15种模式。其他领域比如 Web 开发或嵌入式可能不同，但每个领域都有10-20个核心原则，技术栈只是让实现这些原则更容易而已，所以别太纠结。
  • 人们总推荐骑驴找马是有原因的。如果我对工作不满意，那大概率是时候走了。
  • 我在一些公司交到了可以维系一生的好朋友。但我不要求每份工作都必须如此。有些公司我没和同事成为朋友，但也干得很开心；而有些公司我交到了好友，却依然干得不开心。
  • 我学会了和经理坦诚沟通。不是毫无保留，但足够让我在工作中做真实的自己。他能拿我怎样？开除我？我两周内就能找到新工作。
  • 如果轮值 on-call 时，一个季度里被凌晨两点叫醒超过一次，那绝对是出大问题了。不解决这问题，我就解决掉这份工作。
  • 再倒一杯。
  • 好经理的特质和好工程师的特质有很多重合之处。
  • 刚入行时，我对技术、编程和计算机科学充满迷恋。现在，我已经无感了。
  • 好代码是初级工程师能看懂的代码。更好的代码是大一计算机新生能看懂的代码。最好的代码是压根不写代码。
  • 工程师最被低估的技能是写文档。妈的，谁来教教我怎么写好文档啊？说真的，如果有靠谱的课程推荐，我真愿意花大价钱（比如一千刀，如果保证我能学会写棒棒的文档）。
  • 和上面相关：写好变更提案是一项非常重要的技能。
  • 几乎所有的”圣战”（比如 vim 对 emacs，Mac 对 Linux 等等）都无关紧要……除了下面这一条。
  • 年纪越大，我越欣赏动态语言。妈的，我说出来了。不服来战。
  • 如果我开始觉得自己是房间里最聪明的人，那就是时候离开这个环境了。
  • 我不明白为什么全栈 Web 开发工程师薪水那么低。说真的，他们底薪就该拿到五十万刀一年。妈的，他们既要懂前端又要懂后端，还要懂不同浏览器的工作原理、网络、数据库、缓存、网页端和移动端的区别……而且天啊，公司又想用新框架了？说真的，为什么 Web 开发工程师薪水这么低？
  • 我们应该多招点实习生，他们太棒了。那些精力充沛的小家伙们带着他们的想法。更棒的是他们能提出质疑或批评。我爱实习生。
  • 抿一口。
  • 别见你的偶像。我花了五千刀参加我偶像的一门课。他是个才华横溢的人，但课程结束时我意识到，他其实也和我们一样，在摸索中前进。
  • 技术栈很重要。好吧，我刚说了技术栈不重要，但听我解释。如果你听到 Python 开发者和 C++ 开发者，你想到的是完全不同的东西，对吧？这是因为某些工具就是特别擅长某些工作。如果你不确定想做什么，就学 Java 吧。这是个烂语言，但几乎啥都能干。
  • 最伟大的编程语言是 Lisp。我该去学学 Lisp。
  • 对初学者来说，最赚钱的编程语言是 SQL。去他妈的其他语言。如果你只会 SQL，你也能赚大钱。薪资专员？可能5万刀。会 SQL 的薪资专员？9万刀。大公司里有点组织能力的普通员工？4万刀。有点组织能力还会 SQL 的普通员工？自称项目经理，拿15万刀。
  • 测试很重要，但测试驱动开发（TDD）简直是个邪教。
  • 舒适的政府工作并不像传说中那么好，至少对职业早期到中期的工程师来说。确实，12万刀+福利+养老金听起来不错，但你会出卖灵魂，去搞那些深奥的专有技术。非常尊重政府工作人员，但说真的，那些地方工程师平均年龄50岁以上是有原因的。此建议不适用于政府承包商。
  • 第三方招聘人员是吸血鬼。但是，如果你找到一个好的，一定要认真发展好关系。他们能帮你启动职业生涯。怎么判断是不是好的？如果一个人做第三方招聘超过三年，那他大概率不怎么样。好的通常都去大公司当招聘人员了。
  • 期权要么一文不值，要么让你成为百万富翁。除非工程团队人数超过100人，否则它们大概率一文不值。那样的话，也许这个十年内它们会值点钱。
  • 在家工作太他妈爽了。但不能白板讨论很蛋疼。
  • 我从没在FAANG（脸书、苹果、亚马逊、奈飞、谷歌）工作过，所以不知道我错过了什么。但我招聘（以及拒绝）过来自FAANG的工程师，他们也不知道自己在干嘛。
  • 我的自我价值不是由我的总薪酬决定或与之相关的。资本主义是衡量自我价值的糟糕方式。
  • 经理的权力比你想象的要小。小得多。如果你曾想，为什么XYZ经理不开掉某个人，那是因为他们做不到。
  • 头衔大多不重要。什么”首席杰出骨干工程师”之类的，管他呢。你做了什么，取得了什么成就。人们只关心这个。
  • 说到头衔：职业生涯早期，头衔往上升是好事。初级到中级，中级到高级，高级到主管。职业生涯后期，头衔往下降是好事。这样你可以拿着同样的薪酬，然后通过晋升获得加薪。换句话说，早期（<10年）头衔上升好，因为它让你增长技能和职责。后期，头衔下降好，因为它让你增长薪水。
  • 存满你的401k退休账户。
  • 对所有人都友善。不是因为这对你职业有利（虽然确实有利），而是因为友善本身就有回报。
  • 如果过去一个月我没从初级工程师或实习生那里学到点什么，那一定是我没留意。
  • 靠，酒喝完了。
  • 花钱上课、买书、参加会议是值得的。我参加过一些会议，几门1500刀的课，买过很多书，还订过阅。值。这样我能更好地装成知道自己在干嘛的样子。
  • 说真的，为什么 Web 开发工程师薪水不高？他们啥都得懂啊！！！
  • 腕管综合征和背痛不是开玩笑的。现在花一千刀买好设备值当。
  • 我共事过最聪明的人是个数学博士。我从他那儿学到了太多。希望他一切安好。
  • 高中时，有个女孩是我的好朋友。我们聊天、出去玩，几年间分享了很多私事。后来有传言说我喜欢她，或者我们在交往之类的。她对此反应不佳，开始不理我。那感觉真不好受。我猜这大概相当于现在的”已读不回”。但我并不希望她不好，希望她一切顺利。很抱歉我当时没处理好。
  • 八年级时交了个女朋友，后来我不喜欢她了但又不想提分手，就开始冷落她。我他妈太混蛋了。对不起，莉娜。
  • 你知道当软件工程师最棒的一点是什么吗？你可以遇到和你思维方式一样的人。不一定是喜欢同样的运动、电视剧之类的兴趣。而是他们思考问题的方式和你一样。这挺酷的。
  • 科技行业女性太少了。这真是个该死的行业。需要改变。我一直努力鼓励和帮助机构里的女性工程师，但我不知道还能做些什么。
  • 黑人工程师也是。这他妈怎么回事？
  • 我真正开始讨厌一门语言或技术，通常是在开始深入了解它之后。另外，我觉得一个技术是好的，如果我虽然讨厌它，但同时会推荐给客户。去他妈的 Jenkins，但我觉得推荐给新客户不算软件渎职。
  • 话虽这么说，git 很烂，但我没得选只能用。还有，图形化 git 工具都见鬼去吧，给我命令行就好。大概就7个命令需要记，其他都能谷歌。
  • 既然我搞数据，我就分享点数据领域的经验教训。去他妈的 pandas。
  • 我的工作更轻松，因为团队里有半技术型的分析师。说是半技术型，因为他们懂编程但不懂软件工程。这是好事，如果他们觉得什么东西不合理，那说明设计可能很糟糕。我爱团队里的分析师们；他们帮助我成长，比最才华横溢的工程师给我的帮助还大。
  • 深色模式很棒，直到你被迫用浅色模式（某些网页或不支持的应用）。所以我用浅色模式。
  • 我对安全的了解足够让我知道自己对安全狗屁不通。
  • 靠，酒真喝完了。
  • 做好工程师意味着懂最佳实践。做高级工程师意味着知道何时该打破最佳实践。
  • 如果人们开始试图给 bug 或故障追责，那就是时候走人了。
  • 很多进步的公司，尤其是初创公司，总说要”展现真实的自我”。好吧，如果你的真实自我就是喜欢看黄片呢？所以，在工作和个人生活之间保持界限是健康的。
  • 我喜欢和同事在欢乐时光喝酒。但我更愿意把时间花在孩子、家人或朋友身上。
  • 我见过最伟大的领导力展示，是我的领导为一个100%是我的错的失误承担了责任。你绝对相信我会为她赴汤蹈火。
  • 同样地，我有幸共事过的最好的领导，都尽力既为我的观点辩护，也向我解释与我观点冲突的其他意见。我正努力成为像他们那样的人。
  • 去他妈的业余项目。如果你喜欢做，很好！就算我有时间做业余项目，我也忙着在 Reddit 上发醉酒帖子呢。
  • 算法和数据结构重要——但有个限度。没见药剂师面试考有机化学的琐碎知识。我们行业的面试流程真他妈有问题。
  • 妈的，那些运维的家伙们真他妈聪明。至少他们薪水还不错。
  • 做我喜欢的事不重要。做我不讨厌的事更重要。
  • 我离产品越近，离驱动收入越近，我就越感觉被重视，无论我的工作技术性有多强。即使在最进步的公司也是如此。
  • Linux 很重要，即使我当初全在 Windows 环境下工作。为什么？因为我后来还是在 Linux 下工作了。真庆幸那些周末我折腾安装 Arch Linux。
  • 我学会了警惕那些模糊的流行词，比如”大数据”。他妈什么算”大”数据？我用 Spark 和 Kafka 处理过每10分钟流式传输的1万行数据，也用 Python 和 MySQL 处理过每小时批处理10亿行数据。这些标签都他妈见鬼去吧。
  • 不是所有好工作都在硅谷。但很多确实在。

最后，如果你真想伤害我，别点踩，我不在乎。直接忽略这个帖子就好。没有什么比我写了长文却没人回应更让我难过的了。所以如果你讨厌这个帖子，就无视它吧。

关于编程语言

  • 我曾经讨厌一门编程语言（C#），直到开始使用它。现在我依然讨厌它，但觉得它有用。
  • 后来我开始讨厌一门编程语言（C#），离开了它又回来。哇，这门语言真的进步了好多。
  • 函数式语言最棒的地方在于函数是一等公民，而且所有其他程序员也都知道这一点。
  • 无论一门语言有多伟大或多优越，如果没人用，那就毫无意义。
  • 学习一门语言并不难。难的是学习它的生态系统。

关于同事

  • 结对编程很棒，只是太耗时——而公司通常不愿意花这个时间。
  • 与聪明的工程师共事让我成了更好的码农。与聪明的非技术同事共事让我成了更好的工程师。
  • 不要在工作时间（朝九晚五）之外工作。除非是你自己想，因为你有个超棒的项目而且正干得起劲。那种感觉真他妈爽。
  • 跨团队的欢乐时光和社交时间，99%都只是放松和了解同事。这很酷。偶尔，那1%会涉及一个关键项目和一段关键代码，你会很高兴在社交场合提到了工作，否则可能就出大乱子了。我并不是说因此就该在工作之外和其他团队混在一起。我只是想建立联系。但这他妈的确实是个不错的额外好处。

关于远程工作

  • 如果公司一半远程一半现场，重要的是要判断远程员工是否没有被当作二等公民。如果重大决策都是在”茶水间”做出的，那么最好尝试改变公司文化（很难），或者跳槽到一家将远程员工作为一等公民的公司。
  • 远程工作的第二大缺点是没法用白板。
  • 远程工作的第一大缺点是很难向同事学习。除非我（a）有足够的自信和魄力去提问，并且（b）公司有一种远程员工与现场员工平等的文化，否则我认为我职业生涯头5年在现场工作是最好的。

关于技术

  • 每个人都知道技术会变。过去10年的技术格局发生了巨大变化。但基础的东西变化不大，尤其是适用于我所在领域的基础知识。
  • Hacker News 和 /r/programming 只适合获取大致想法和保持更新。评论几乎毫无价值。
  • 有很多嗓门大的业余爱好者对技术有强烈的看法。甚至有些业余爱好者还在”受人尊敬”的期刊和博客上发表文章。我发现最好是了解这些传闻，但要自己动手搞清楚。
  • 我在一家尖端初创公司工作，我们并没有使用那家在ABC的尖端科技公司展示的最新XYZ技术。结果发现，他们通常展示的只是他们工程部门的一小部分，而且他们大部分人使用的技术和我们一样。
  • 话虽这么说，留意迹象很重要。如果你想从事现代技术工作，而你的公司大部分开发仍然在用 jQuery，也许是时候重新评估了。

关于数据工程

  • 妈的，反正我是数据工程师，不如给出更具体、更有针对性的建议/经验。
  • SQL 是王道。像 MySQL、Postgres、Oracle、SQL Server、SQLite 这样的数据库仍然占据统治地位。即使你使用新技术，大部分知识也都能迁移。
  • 大多数公司并不做流处理。这既难又复杂。如果你从业10年了还不知道如何处理每秒1万条的记录，别担心，还是有工作适合你的。
  • Airflow 很烂，没错。是有其他产品，但妈的，Airflow 难道不是用得最广的吗？
  • 机器学习项目很容易失败。它们复杂且难以实施。不相信？给他妈的机器学习模型写单元测试有多容易？是吧。
  • 我们这个领域还很新。没有关于数据工程的好书，只管去”做”就行了。没法通过训练营之类的东西学会。等我们所有人都他妈搞明白自己在做什么之后，这种情况10年内可能会改变。

关于生活

  • 人都会死。你希望你的代码成为你的遗产吗？如果是，那就花大量时间在上面，因为那是你他妈的遗产，加油！但如果你像我一样，你的遗产是身边的家人、朋友和生活中的人，而不是你写的代码。所以别太纠结于此。
  • 好人会写烂代码。聪明人会写烂代码。好的码农和好的工程师也会写烂代码。别让代码质量成为你自我价值的因变量。
  • 我进入科技和编程领域是因为科技曾是我的爱好。现在我的爱好和工作一样，而工作毁了我的爱好。所以现在，如果我想享受科技的乐趣，我就得放弃我的爱好。或者我得接受科技不再是我的爱好，并寻找新的爱好。
  • 编程和计算机科学才大概，呃，80年历史？跟任何其他工程学科比比看。是啊，我们集体都不知道自己他妈的在做些什么。
  • 我赚的钱相当不错。要心存感激和欣赏。还有，要存钱。

其他杂事

  • 我构建过被多个团队和人员使用多年的大型平台和库。但不知为何，我对自己写的代码最自豪的，却是那个只有我自己在用的小脚本。
  • 我职业生涯中最自豪的成就是帮助其他人把工作做得更好。这可能是因为我注定要成为人员经理，所以这对其他人可能没什么帮助。
  • 当初我找工作时，更新了我的LinkedIn。我收到一堆垃圾回复，然后就把它删了。现在我使用LinkedIn来为我的公司寻找其他候选人。底线是，LinkedIn上噪音太多。我发现它有价值，只是因为现在我的部分工作就是在贡献这种噪音。
  • 有一次，我在大学里发现一个女孩喜欢我。我不相信，因为我自尊心很低，但后来她约我出去。我告诉她我没兴趣，尽管她真的很酷。那是我人生中最自豪的时刻之一，因为我在19岁时就足够成熟，能以成熟的方式说”不”。
  • /r/cscareerquestions 简直就是自我中心和错误信息的粪坑，我都不知道该怎么办。就像，他妈什么鬼。我想摇醒所有那些人，试着向他们解释这个世界到底是怎么运作的，但他们不会相信我。

关于我此刻的大致感受

  • 我喝醉了，而且我平时不喝酒，所以我觉得我说的一切可能都很尴尬或很糟糕。
  • 我强烈认为人们应该存钱和投资。如果你有六位数的薪水，请尽力存满你的401k。
  • 我变成了我一直讨厌的那种人：在职业生涯中从事科技工作，但在现实生活中避开科技。也许这是变老的一部分。
  • /r/ExperiencedDevs 是一个很酷的社区。谢谢版主们。你们得到的赞赏远不及你们应得的。说真的，谢谢。
  • 我大概把我的职业生涯、我的薪水、我的人生都归功于Reddit。Reddit 被骂得很惨，但这里的社区把我从贫困（在加油站工作拿最低工资）中拉了出来，通过学习 Linux、SQL、Python、C# 和其他东西，让我走到了今天。
  • 孩子很棒。我选择不要孩子。为什么？因为我爱孩子，而且我害怕自己会成为一个什么样的父亲。哦靠，这对这里的帖子来说是不是太私人了？
